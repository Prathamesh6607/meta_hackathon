Tier-2 Support Benchmark

Email Triage OpenEnv Playground

We solve customer-support workflow automation: classify incoming emails, apply policy correctly, and resolve issues with tool-based actions.

Problem We Are Solving

Support teams receive large volumes of mixed-intent emails. This environment trains and evaluates agents to triage emails, follow return-policy rules, and execute multi-step resolutions with reliable reward signals.

Pros

  • Chained task pipeline: task_1 output → task_2 input → task_3 input.
  • Deterministic graders with partial-credit rewards (no randomness).
  • HF Space ML assistance with automatic fallback (always completes).
  • Transparent epoch logs, agent policy logs, and per-step traces.
  • Dataset-backed case generation (real support corpus patterns).
  • Full end-to-end pipeline runs with accurate handoff payloads.

Cons

  • Task domain is specific to customer support (not general NLP).
  • HF Space latency affects assisted mode response time.
  • Policy rules are simplified (real support has more edge cases).
  • Synthetic datasets: local corpus patterns may differ from production.
  • Task sizes are small (4 emails, 2-3 steps per task).
  • Inventory/order DB are mocked (not real system integration).
